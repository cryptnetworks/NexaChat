# Security policy

Do not open public issues for suspected vulnerabilities or include secrets, private content, or exploit details in ordinary reports. Until a private reporting channel is configured, contact the repository owner through a private channel listed on the hosting profile and include affected versions, impact, reproduction steps, and suggested mitigations. Do not access other people's data or disrupt services while researching.

Only released versions designated as supported in release notes receive fixes. The project is pre-release and currently provides no supported production version.
